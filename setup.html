安装指南

几分钟内让 OpenClaw 运行起来——有引导脚本、可执行命令，无需猜测。

🧙 安装向导

OpenClaw 安装向导是一个免费的浏览器应用，引导你完成安装、配置、渠道添加和技能安装。选择你的目标和平台，按照清晰的步骤操作，然后下载或复制你需要的脚本或配置。无需注册，无需账号——直接打开向导即可。

🚀 打开安装向导

向导如何工作

打开向导，选择你想做什么：安装 OpenClaw、配置（AI 提供商和认证）、添加渠道（如 Telegram 或 Discord）、或添加技能。对于安装，你需要选择操作系统（macOS、Linux、Windows WSL2、Docker 或一键安装脚本）；对于渠道和技能，你会获得分步说明以及可用的配置或脚本。你可以从页面下载或复制脚本，或使用"在我的机器上运行"让向导将脚本发送到你的 OpenClaw 伴侣。

向导适合谁

  • 首次安装 OpenClaw，想要一个针对你系统的脚本
  • 添加新渠道（消息应用），需要确切的配置和 token 步骤
  • 添加技能，想要一个也重启 OpenClaw 的安装脚本
  • 寻找故障排除帮助，更喜欢从应用内搜索文档
  • 不想手动编辑 openclaw.json：向导生成可合并的渠道 JSON 和 AI 提供商的完整配置片段

为什么使用向导？

  • 一个地方搞定一切——安装、配置、添加渠道、添加技能
  • 为你生成的脚本——获取针对你平台的就绪脚本和渠道配置 JSON
  • 分步说明——每个渠道和技能都有简短清晰的步骤，不会迷路
  • 搜索文档——从应用内打开内置文档搜索，查找安装、配置或故障排除答案
  • 你的机器，你的数据——所有脚本和配置在浏览器中生成；不发送到任何服务器

💻 系统要求

  • 运行时环境：Node.js 22.12.0 或更高版本（包含重要的安全修复；支持 npm、pnpm 或 bun）
  • 支持的平台：macOS、Linux、Windows（WSL2 推荐用于 Windows 用户）
推荐使用一键安装脚本——它会自动安装 Node.js（如需要）和所有依赖项。

⚡ 一键安装

一行命令搞定。安装向导生成针对你操作系统量身定制的脚本，自动处理 Node.js 和依赖项。

macOS / Linux

curl -fsSL https://openclaw.ai/install.sh | bash openclaw onboard

Windows (PowerShell)

iwr -useb https://openclaw.ai/install.ps1 | iex openclaw onboard

Windows (CMD)

curl -fsSL https://openclaw.ai/install.cmd -o install.cmd && install.cmd && del install.cmd openclaw onboard
macOS + Ollama 0.17+？最快路径：ollama launch openclaw --model kimi-k2.5:cloud。一条命令安装和配置一切。参见 Ollama + OpenClaw 教程。

📦 npm 或 pnpm 安装

如果你已经有 Node.js 22.12.0+：

npm i -g openclaw # 或：pnpm add -g openclaw openclaw onboard

🍎 macOS 伴侣应用

下载 macOS 版（菜单栏访问，与 CLI 并行工作）。需要 macOS 14+。

最新版本：GitHub Releases

⚙️ 配置

基础配置

最小配置：~/.openclaw/openclaw.json（较新安装）或 ~/.clawdbot/moltbot.json（旧版兼容）。工作区根目录是 ~/.openclaw/workspace 或 ~/clawd（取决于安装方式）。CLI 是 openclaw；moltbot 和 clawdbot 保留作为兼容别名。

"agent": { "model": "anthropic/claude-opus-4-6" }

语言模型选项与认证

推荐配置：Anthropic Pro/Max 订阅（100/200）配合 Opus 4.6 以增强上下文处理和更好的提示注入攻击防护。

基于 OAuth 的订阅：

  • Anthropic（Claude Pro/Max 计划）
  • OpenAI（ChatGPT Plus/Codex 访问）

虽然平台支持各种语言模型，但 Anthropic Pro/Max 配合 Opus 4.6 是推荐选择。你可以配置自动模型故障转移和认证配置文件轮换（在 OAuth 和 API 密钥方法之间切换）以及备份选项。

💬 消息渠道集成

OpenClaw 支持所有主流消息平台。对于分步指导和使用可合并配置的引导流程，使用安装向导（添加渠道）。

支持的平台

  • WhatsApp — 通过 Baileys 库集成，QR 码认证
  • Telegram — 配置为 Telegram 机器人，使用 grammY 框架
  • Discord — 完整的 Discord 机器人功能，支持斜杠命令
  • Slack — 工作区集成，使用机器人令牌和应用令牌
  • Signal — 安全消息集成，需要安装 signal-cli
  • iMessage — macOS 专属，需要已登录 Messages 应用
  • Microsoft Teams — 通过 Microsoft Bot Framework 集成
  • Google Chat — 工作区集成
  • WebChat — 通过网关 WebSocket 访问的本机 Web 界面

快速设置示例（WhatsApp）

  • 设备认证：openclaw channels login（凭据保存到 ~/.clawdbot/credentials）
  • 配置访问控制：设置 channels.whatsapp.allowFrom 指定授权联系人
  • 群组访问：配置 channels.whatsapp.groups 作为群组白名单；使用 "*" 允许所有群组
注意：WhatsApp 认证需要 QR 码配对。所有凭据本地加密存储。配置目录路径 ~/.clawdbot/ 为早期版本（Clawdbot 和 Moltbot）的向后兼容而保留。

🐳 Docker 安装

docker run -it --rm \ -v ~/.openclaw:/app/data \ -p 18789:18789 \ openclaw/openclaw:latest

官方镜像以非 root 用户运行。额外加固：--read-only --cap-drop=ALL。

📁 工作区结构与技能

OpenClaw 在专用工作区目录中组织所有数据：

  • 工作区位置：~/.openclaw/workspace 或 ~/clawd（可通过 agents.defaults.workspace 配置）
  • 核心提示文件：AGENTS.md、SOUL.md、TOOLS.md
  • 技能存储：~/clawd/skills/<skill-name>/SKILL.md
  • 记忆存储：OpenClaw 自动生成的每日 Markdown 格式笔记

工作区由你系统上的标准文件夹和 Markdown 文件组成。这些可以直接编辑，可以使用 Raycast 等工具搜索，或与 Obsidian 等应用同步。

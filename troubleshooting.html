故障排除

修复常见问题，解决你可能遇到的障碍

大多数 OpenClaw 问题分为几类：网关无法启动、渠道无法连接、或 Agent 没有响应。本文涵盖这些以及通常有效的修复方法。遇到问题先运行 openclaw doctor——它能捕获很多问题。

快速参考 - 常用命令：
  • openclaw restart 或 openclaw gateway restart — 重启网关
  • openclaw doctor --fix — 健康检查并自动修复常见问题（权限、配置、缺失目录）

寻求帮助前：运行 openclaw doctor（或 openclaw doctor --fix），查看 FAQ 了解你的症状，报告 bug 时请包含你的 OpenClaw 版本（openclaw --version）。

⚡ 快速修复

如何重启 OpenClaw / 网关重启

OpenClaw 网关重启——使用以下之一：

  • openclaw restart — 快速重启
  • openclaw gateway restart — 显式网关重启

如果 OpenClaw 作为服务运行：

  • Linux (systemd)：systemctl --user restart openclaw-gateway（服务名称可能不同，用 systemctl --user list-units | grep openclaw 检查）
  • macOS (launchd)：从 OpenClaw 菜单栏应用重启，或 launchctl unload ~/Library/LaunchAgents/ai.openclaw.gateway.plist 然后重新启动

🔧 常见问题

网关无法启动

可能原因：

  • Node.js 版本过旧（需要 ≥22）
  • 端口 18789 已被占用
  • 配置文件有语法错误
  • 缺失依赖项

解决方案：

  • 运行 openclaw doctor 检查配置问题
  • 验证 Node.js 版本：node --version
  • 检查端口是否被占用：lsof -i :18789（macOS/Linux）
  • 审查配置文件：~/.openclaw/openclaw.json 或 ~/.clawdbot/moltbot.json
  • 重新安装：npm install -g openclaw@latest

渠道无法连接

可能原因：

  • 凭据或 token 不正确
  • 缺失环境变量
  • 网络连接问题
  • 渠道特定配置错误

解决方案：

  • 验证凭据正确
  • 对于 WhatsApp，确保已完成配对流程
  • 检查 ~/.clawdbot/credentials 中的存储凭据
  • 验证环境变量设置正确
  • 查看渠道特定文档
  • 启用详细日志：openclaw gateway --verbose

权限错误（macOS）

可能原因：

  • 系统设置中缺失 macOS 权限
  • 未授予屏幕录制权限
  • 辅助功能权限缺失

解决方案：

  • 打开「系统设置」→「隐私与安全性」
  • 授予必要权限：屏幕录制、辅助功能、完全磁盘访问（如果需要）
  • 在工具调用中设置 needsScreenRecording: true
  • 授予权限后重启网关

模型认证问题

可能原因：

  • API 密钥无效
  • OAuth token 过期
  • 模型配置错误
  • 速率限制或配额超出

解决方案：

  • 验证 API 密钥或 OAuth token 有效
  • 检查 ~/.clawdbot/moltbot.json 中的模型配置
  • 测试认证：openclaw agent --message "test"
  • 检查提供商仪表板的速率限制或配额
  • 配置模型故障转移以提高可靠性

技能不工作

可能原因：

  • 技能文件语法错误
  • 缺失依赖项
  • 技能配置错误
  • 权限问题

解决方案：

  • 检查 ~/clawd/skills/ 中的技能文件
  • 审查技能日志中的错误
  • 验证技能依赖项已安装
  • 手动测试技能
  • 如需要，重新创建技能

错误：绑定的 Chrome 扩展缺失——重新安装 OpenClaw

错误信息：error: bundled chrome extension is missing. reinstall openclaw and try again.

修复：

  • 重新安装 OpenClaw：npm install -g openclaw@latest 或 pnpm add -g openclaw@latest
  • 重启终端，如果使用守护进程则运行 openclaw onboard --install-daemon
  • 重启网关：openclaw gateway restart
  • 如果错误持续，完全退出 OpenClaw 应用（macOS 菜单栏）或停止服务，然后重新启动

无法到达 OpenClaw 浏览器控制服务

错误信息：can't reach the openclaw browser control service. start (or restart) the openclaw gateway 或 chrome extension relay is running, but no tab is connected

可能原因：

  • Chrome/Chromium 未安装
  • 浏览器控制未启用
  • Linux 特定的显示问题
  • Chrome 扩展未连接到标签页
  • 网关未运行

解决方案：

  • 验证 Chrome/Chromium 已安装
  • 在配置中启用浏览器控制：browser.enabled: true
  • 在 Linux 上，确保显示可用（X11 或 Wayland）
  • 检查浏览器控制 URL 配置
  • 点击标签页上的 OpenClaw Chrome 扩展图标以附加
  • 重启网关：openclaw gateway restart

无输出 / 无响应

问题：OpenClaw 无输出或不响应消息

可能原因：

  • 网关未运行
  • 渠道未连接
  • 模型认证失败
  • 配置错误
  • 网络连接问题
  • 工具未启用 — Agent 可以聊天但无法读取文件、运行命令或使用浏览器（见下方）

解决方案：

  • 检查网关状态：openclaw gateway status
  • 重启网关：openclaw gateway restart
  • 运行诊断：openclaw doctor --fix
  • 检查渠道连接：openclaw channels list
  • 验证模型配置和 API 密钥
  • 检查日志：openclaw gateway logs
Agent 回复但不运行工具（读取文件、运行命令、浏览器）？你的 Agent 可能没有启用工具。OpenClaw 可能默认使用 tools.profile: "messaging"，只允许发送消息——无文件访问、无 exec、无浏览器。

修复：在配置中设置 "tools": { "profile": "full" }（或如果是编码场景用 "coding"），然后重启网关。如果需要 Agent 无需每次都请求批准就能运行命令（例如在 Telegram 上没有确认弹窗），在工具下添加 "exec": { "security": "full", "ask": "off" }。先设置 profile；exec.ask 只在工具启用时适用。
账单 / API 错误（2026.2.19+）：当模型或提供商返回账单或配额错误时，OpenClaw 现在在消息中包含活动模型（例如 OpenAI (gpt-5.3)），这样你可以看到哪个密钥或账户需要充值。

网关 Token 缺失 / 未授权（1008）

错误信息：disconnected (1008): unauthorized: gateway token missing 或 gateway token missing (open a tokenized dashboard url or paste token in control ui settings)

解决方案：

  • 打开包含 token 的令牌仪表板 URL
  • 或在控制 UI 设置中粘贴 token
  • 使用 token 重启网关：openclaw gateway --token YOUR_TOKEN
  • 检查网关配置中的 token 设置
  • 生成新 token：openclaw doctor --generate-gateway-token，然后打开令牌仪表板 URL：openclaw dashboard --no-open
注意（2026.2.19+）：如果你没有设置网关认证，OpenClaw 现在可以在启动时自动生成并持久化 gateway.auth.token，以便控制 UI 和客户端可以连接。你仍可以显式设置 gateway.auth.mode: "none" 用于本地仅回环设置。

断开连接（1008）：需要配对

错误：disconnected (1008): pairing required 或 pairing required openclaw

控制 UI 或客户端必须在连接前完成配对。打开 OpenClaw 仪表板（或你的令牌 URL），进入设置/设备（或等效位置），完成配对步骤。对于 DM 渠道（如 Telegram、WhatsApp），使用 openclaw pairing approve <channel> <code> 批准新联系人。

配对 CLI（2026.2.21）：省略时 pairing list 和 pairing approve 默认为唯一的可用配对渠道，无需猜测渠道参数（对纯 TUI 设置很有用）。

管理配对设备（2026.2.19+）：使用 openclaw devices remove 移除特定设备，或 openclaw devices clear --yes [--pending] 清除配对条目并可选择拒绝待处理请求。配置也支持 device.pair.remove 用于清理流程。

网关已在运行 / PID 锁超时

错误信息：gateway already running pid lock timeout

解决方案：

  • 检查运行中的网关进程：ps aux | grep openclaw（Linux/macOS）
  • 如果进程未运行，删除锁文件：rm ~/.clawdbot/gateway.lock
  • 终止任何卡住的进程：killall openclaw（Linux/macOS）
  • 重启网关：openclaw gateway restart

🏥 健康检查

运行 openclaw doctor 执行全面健康检查：

  • 检查配置问题
  • 发现危险的/配置错误的 DM 策略
  • 验证网关状态
  • 检查更新
  • 验证渠道配置
  • 检查工作区完整性
openclaw doctor [--fix]

使用 openclaw doctor --fix 自动修复常见问题，如权限问题、缺失目录和配置错误。

📋 日志

启用详细日志以诊断问题：

openclaw gateway --verbose

或在配置文件中设置 verboseLevel 以获得持久详细输出。

日志位置：

  • 网关日志：检查终端输出或系统日志
  • 渠道日志：工作区中的按渠道日志
  • Agent 日志：工作区中的会话日志

🔒 网关锁

如果网关看起来被锁定或无法启动：

  • 检查锁文件：~/.clawdbot/gateway.lock
  • 验证没有其他网关实例正在运行
  • 如果网关进程未运行，删除锁文件：rm ~/.clawdbot/gateway.lock
  • 重启网关

⚙️ 后台进程

如果网关作为后台服务运行：

  • macOS：检查 launchd：launchctl list | grep openclaw
  • Linux：检查 systemd：systemctl --user status openclaw-gateway
  • 查看日志：journalctl --user -u openclaw-gateway（Linux）或检查 Console.app（macOS）
  • 重启服务：systemctl --user restart openclaw-gateway（Linux）

🐧 Linux 浏览器故障排除

在 Linux 上，浏览器控制可能需要额外设置：

  • 安装 Chrome/Chromium：sudo apt install chromium-browser（Debian/Ubuntu）
  • 确保显示服务器正在运行（X11 或 Wayland）
  • 必要时设置 DISPLAY 环境变量
  • 如果没有可用显示器，使用无头模式

🆘 获取帮助

如果你仍然遇到问题：

  • 运行 openclaw doctor 并审查所有输出
  • 启用详细日志并捕获错误信息
  • 查看配置指南了解设置
  • 查看渠道指南了解渠道特定问题
  • 检查工作区文件中的错误
  • 打开安装向导并使用「修复问题」从应用内搜索文档
